## 📋 完整函数签名列表

### 1. shortenPath
```cpp
void FastExplorationManager::shortenPath(vector<Vector3d> &path)
```
- **输入**: `path` (vector<Vector3d>&) - 待优化路径
- **输出**: 修改输入path，移除冗余点
- **目的**: 路径简化

### 2. findNextCluster ⭐⭐⭐⭐⭐
```cpp
void FastExplorationManager::findNextCluster(
    const Vector3d &cur_pos,
    const Vector3d &cur_vel,
    const Vector3d &cur_yaw,
    vector<checkPoint> &check_tour,
    Eigen::Vector3d &next_cluster_pos,
    const bool neighbor)
```
- **输入参数**:
  1. `cur_pos` (Vector3d) - 当前位置
  2. `cur_vel` (Vector3d) - 当前速度
  3. `cur_yaw` (Vector3d) - 当前偏航角
  4. `neighbor` (bool) - 是否考虑邻近聚类
- **输出参数**:
  1. `check_tour` (vector<checkPoint>&) - 检查点路线
  2. `next_cluster_pos` (Eigen::Vector3d&) - 下一聚类位置
- **目的**: 在多个frontier聚类中选择最优目标

### 7. solveTSP ⭐⭐⭐⭐⭐
```cpp
void FastExplorationManager::solveTSP(
    const Eigen::MatrixXd &cost_matrix,
    const TSPConfig &config,
    vector<int> &result_indices,
    double &total_cost)
```
- **输入参数**:
  1. `cost_matrix` (Eigen::MatrixXd) - N×N成本矩阵
  2. `config` (TSPConfig) - 求解配置
- **输出参数**:
  1. `result_indices` (vector<int>&) - 最优路线索引
  2. `total_cost` (double&) - 路线总成本
- **目的**: 通用TSP求解接口

**TSPConfig结构体**:
```cpp
struct TSPConfig {
    string problem_name_;     // 问题名称
    int dimension_;           // 问题维度
    bool skip_first_;         // 跳过首点
    bool skip_last_;          // 跳过末点
    int result_id_offset_;    // ID偏移
};
```

## 🎯 函数使用场景

### 感知+规划循环
```
每个规划周期:
  1. updateFrontierStruct(pos)          ← 更新frontier状态
  2. getFrontierDivision(...)           ← 获取聚类
  3. findNextCluster(...)               ← 选择下一聚类 ⭐
     ├─ updateInertialTour(...)         ← 计算惯性成本
     └─ findLocalTour(...) + solveTSP   ← TSP求解
  4. shortenPath(path)                  ← 优化路径
  5. clearExplorationData()             ← 重置状态
```

## 🚀 核心创新函数组合

### 组合1: 聚类选择策略
```cpp
findNextCluster() 
  + updateInertialTour()
  + solveTSP()
```
→ 实现Fuel独特的**多聚类动态选择**

### 组合2: TSP求解框架
```cpp
findLocalTour() 或 findGlobalTour()
  + solveTSP()
```
→ 提供**灵活的路线优化**

### 组合3: 路线优化管道
```cpp
findLocalTour()
  + shortenPath()
  + clearExplorationData()
```
→ 实现**完整的规划周期**
