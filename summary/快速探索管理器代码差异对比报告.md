## 🎯 4. 核心算法函数改动

### ✨ **最大改动**: 主规划函数重命名和重构

#### Star-searcher
```cpp
int FastExplorationManager::planExploreMotion(
    const Vector3d& pos, const Vector3d& vel, const Vector3d& acc, const Vector3d& yaw)
```

#### Fuel
```cpp
int FastExplorationManager::planExploreMotionCluster(const Vector3d &pos,
                                                     const Vector3d &vel,
                                                     const Vector3d &acc,
                                                     const Vector3d &yaw)
```

**改动原因**: Fuel版本采用**集群化策略**处理frontier

### 内部逻辑差异

#### Star-searcher (简单全局路由)
- 获取所有frontier的viewpoint
- 为所有viewpoint创建单一全局TSP
- 返回单个next_yaw

```cpp
double next_yaw;
if (ed_->points_.size() > 1) {
    findGlobalTour(pos, vel, yaw, indices);
    // 进行局部优化处理...
}
```

#### Fuel (集群化处理)
- 获取frontier分割聚类
- 确定下一个聚类
- 返回vector<double> next_yaw用于多步规划

```cpp
vector<vector<Eigen::Vector3d>> division_clusters;
frontier_finder_->getFrontierDivision(division_clusters);
Eigen::Vector3d next_cluster_pos;
if (division_clusters.size() > 1) {
    findNextCluster(pos, vel, yaw, ed_->local_tour_, next_cluster_pos, neighbor);
} else {
    // 单集群处理...
}
vector<double> next_yaw;
if (ed_->local_tour_.size() > 1) {
    // 处理多步规划...
}
```

## 📊 5. 数据结构和存储差异

### Star-searcher
- `ed_->global_tour_` - 全局路线
- `ed_->frontier_boxes_` - frontier边界框
- `ed_->dead_frontiers_` - 死frontier
- `ed_->refined_tour_` - 细化后的路线

### Fuel
- `ed_->global_tour_` - 全局路线
- `ed_->local_tour_` - **新增**: 局部路线
- `ed_->grid_tour_` - **新增**: 栅格路线
- `ed_->inertia_tour_` - **新增**: 惯性路线
- `division_clusters` - **新增**: 聚类分割

**差异分析**:
- Fuel采用多层次路线表示(全局/局部/栅格/惯性)
- 支持基于聚类的分层规划
