### 1️⃣ **planExploreMotion → planExploreMotionCluster** (重命名)

#### Star-searcher 版本
```cpp
int FastExplorationManager::planExploreMotion(
    const Vector3d& pos,
    const Vector3d& vel,
    const Vector3d& acc,
    const Vector3d& yaw)
```

#### Fuel 版本
```cpp
int FastExplorationManager::planExploreMotionCluster(
    const Vector3d &pos,
    const Vector3d &vel,
    const Vector3d &acc,
    const Vector3d &yaw)
```

| 项目 | Star-searcher | Fuel |
|------|---------------|------|
| **函数名** | `planExploreMotion` | `planExploreMotionCluster` |
| **策略** | 全局TSP规划 | 聚类化分层规划 |
| **返回值** | int (SUCCEED/FAIL) | int (SUCCEED/FAIL) |

| 方面 | Star-searcher | Fuel |
|------|---------------|------|
| Frontier处理 | 获取所有viewpoint | 获取frontier分割聚类 |
| 路线求解 | 单一全局TSP | 聚类选择 + 局部/全局TSP |
| 返回值 | `double next_yaw` | `vector<double> next_yaw` |

## 🎯 函数调用流程

### 当frontier分割为多个聚类时 (推荐路径)

```
planExploreMotionCluster
  ↓
updateFrontierStruct
  ↓
getFrontierDivision (多个聚类)
  ↓
findNextCluster ← 使用惯性成本或TSP成本比较
  ├─→ updateInertialTour (计算惯性成本)
  │    ↓
  │    findLocalTour (局部TSP)
  │     ↓
  │    solveTSP
  │
  └─→ findLocalTour (普通TSP)
       ↓
      solveTSP
  ↓
返回: next_cluster_pos, check_tour
```

### 当frontier为单个聚类时 (简化路径)

```
planExploreMotionCluster
  ↓
updateFrontierStruct
  ↓
getFrontierDivision (单个聚类)
  ↓
findGlobalTour
  ↓
solveTSP (全局TSP)
  ↓
返回: 全局路线索引
```

## 🔍 核心创新

### Fuel的三大创新函数

1. **findNextCluster** - 多聚类选择策略
   - 支持惯性规划 (考虑运动连续性)
   - 支持普通TSP优化 (成本最优)
   - 支持邻近聚类优先

2. **updateInertialTour** - 惯性路线规划
   - 考虑特征点距离
   - 实现成本偏移调节
   - 执行局部TSP优化

3. **solveTSP** - 统一TSP求解接口
   - 灵活的配置系统
   - 支持多类型问题
   - 自动文件管理
